# Luma Studio - Fase 8: Checklist QA general

Fecha: 2026-06-18

## Objetivo

Checklist de validacion antes de considerar la primera version lista para demo con cliente.

## QA publico

- Home tiene hero visual.
- Header funciona desktop/mobile.
- Proyectos cargan.
- Filtros funcionan.
- Detalle de proyecto carga.
- Albumes cargan.
- Masonry no rompe layout.
- Lightbox funciona con mouse, teclado y mobile.
- Cortos muestran poster y CTA.
- Sobre/contacto muestran contenido correcto.

## QA admin

- Login correcto.
- Login incorrecto.
- Dashboard carga.
- CRUD proyecto.
- CRUD album.
- Media upload.
- Editar alt/caption.
- Asociar media.
- Reordenar album.
- Publicar/despublicar.
- Settings.

## QA responsive

Validar:

- 390px.
- 768px.
- 1024px.
- 1440px.

Criterios:

- Sin overflow horizontal.
- Texto no se corta.
- Botones tactiles.
- Imagenes no aplastadas.
- Admin usable en laptop.

## QA performance

- Hero estable.
- Imagenes lazy.
- No cargar embeds de video innecesarios.
- Build sin errores.
- Lighthouse razonable.

## QA accesibilidad

- Alt obligatorio.
- Focus visible.
- Labels en formularios.
- Lightbox con teclado.
- Contraste aceptable.
